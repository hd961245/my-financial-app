---

## 1. 產品概述 (Product Overview)

### 1.1 產品定位

金融資訊整合平台是一個面向**個人投資者**的私人財務儀表板，整合股市報價、AI 分析、自訂新聞源、紙上交易模擬帳戶與 Google Sheets 自選股追蹤，讓使用者在單一介面管理並分析自己的投資組合。

### 1.2 核心價值主張

- **資訊整合**：一站式彙整市場指數、財經新聞、個股健康度與自訂訂閱源。
- **AI 加持**：透過 GPT-4o 與 Gemini 雙模型提供個股分析、技術指標解讀與社群情報過濾。
- **紙上交易**：模擬真實帳戶買賣流程，記錄績效，無資金風險。
- **客製彈性**：支援 Google Sheets 自選股清單連動，並可訂閱任意 RSS/HTML 資料源。

### 1.3 使用語言

繁體中文 (Traditional Chinese) 為主要介面語言，部分技術標籤保留英文。

## 3. 產品目標與成功指標 (Goals & Success Metrics)

### 3.1 產品目標

1. 讓使用者每天在 **< 5 分鐘**內完成早盤市場瀏覽。
2. 讓使用者透過 AI 輔助，在 **< 30 秒**內完成單一個股技術分析。
3. 提供完整的紙上交易記錄，支援績效回顧。

### 3.2 成功指標

| 指標 | 目標 |
|------|------|
| 市場指數載入時間 | < 3 秒（4 支指數並行請求） |
| AI 個股分析回應時間 | < 10 秒 |
| 每週 AI 健檢報告產生時間 | < 45 秒（10 支股票） |
| 批次匯入成功率 | > 95%（符合格式的 CSV/Excel） |
| RSS 自訂源聚合成功率 | > 90%（每個來源 10 秒 timeout） |

## 4. 功能規格 (Feature Specifications)

### 4.1 市場動態 (Market Overview)

**目的：** 提供每日開盤前快速掌握全球市場狀況。

#### 功能項目

| 功能 | 描述 |
|------|------|
| 市場指數卡片 | 即時顯示 S&P 500（^GSPC）、台灣加權指數（^TWII）、NASDAQ（^IXIC）、美元/台幣（TWD=X）的價格與漲跌幅 |
| 自動輪詢 | 每 60 秒自動更新指數資料 |
| 資產配置圓餅圖 | 依投資分類顯示目前持股的市值佔比，連動至 Portfolio 模組的即時報價 |
| 財經新聞 | 從 Yahoo Finance RSS 取得最新 5 則新聞，包含標題、來源、時間與縮圖 |
| 個人化動態牆 | 聚合使用者自訂 RSS/HTML 資料源的最新消息 |

#### 使用者故事

- 作為投資者，我希望在開啟儀表板時**立即看到**主要指數的當前報價，不需要手動刷新。
- 作為投資者，我希望看到圓餅圖反映我各類別資產的**即時佔比**，幫助我判斷是否需要再平衡。

#### 邊界條件

- 指數資料無法取得時，卡片顯示 "Loading..."，不顯示錯誤碼。
- 圓餅圖資料為空時，顯示「無資產或分類資料」提示。

**目的：** 快速判斷各產業的法人買賣力道與市場情緒。

#### 功能項目

| 功能 | 描述 |
|------|------|
| 產業熱力圖 | 以色塊展示 6 大產業（半導體、金融、航運、生技、AI 概念、綠能）的漲跌幅，綠色代表買超、紅色代表賣超 |
| 法人連續買超榜 | 顯示法人連續淨買超的前幾名個股，含成交量 |
| 法人連續賣超榜 | 顯示法人連續淨賣超的前幾名個股，含成交量 |

#### 使用者故事

- 作為投資者，我希望在 10 秒內判斷今日哪個產業有法人資金流入，輔助選股方向。

### 4.3 個股健康度 (Stock Health Analyzer)

**目的：** 單一個股的技術面與基本面快速健檢。

#### 功能項目

| 功能 | 描述 |
|------|------|
| 股票搜尋 | 輸入股票代號（如 AAPL、2330.TW、台積電），支援中文轉換 |
| 即時報價 | 顯示當前股價與日漲跌幅（% 與絕對值） |
| 技術指標 | 5MA、10MA、20MA、60MA、240MA，並標示當前股價與各均線的相對位置 |
| 動能評分 | 依均線排列判斷：強力多頭 / 多頭 / 中性 / 空頭 / 強力空頭 |
| 籌碼佈局 | 法人持股比例、內部人持股、散戶浮動比估算 |
| 財務指標 | EPS（TTM）、Forward P/E、營收成長率、毛利率、營業利潤率 |
| 自動刷新 | 瀏覽個股期間每 10 秒更新一次 |

#### 使用者故事

- 作為投資者，我希望輸入一個股票代號後，在 5 秒內看到完整的技術面健康評分，幫助我決定是否進一步研究。
- 作為台灣投資者，我希望能直接輸入中文股票名稱（如「台積電」），系統自動轉換為正確代號。

### 4.4 投資組合追蹤 (Portfolio Tracker)

**目的：** 完整的模擬帳戶管理，記錄交易與追蹤績效。

#### 4.4.1 帳戶管理

| 功能 | 描述 |
|------|------|
| 帳戶餘額 | 顯示目前可用現金 |
| 累計存入 | 歷史存入總額 |
| 出入金 | 存入/提出現金，含完整流水帳記錄 |
| 餘額保護 | 買入時若餘額不足，回傳明確錯誤訊息 |

#### 4.4.2 績效指標（4 卡片）

| 卡片 | 計算邏輯 |
|------|---------|
| 總市值 | Σ (持有股數 × 即時報價) |
| 未實現損益 | 總市值 - Σ (持有股數 × 平均成本) |
| 已實現損益 | Σ (賣出均價 - 買入均價) × 賣出股數 |
| 帳戶總損益 | 未實現損益 + 已實現損益 |

#### 4.4.3 投資分類管理

| 功能 | 描述 |
|------|------|
| 建立分類 | 自訂命名（如「台股科技」、「美股 ETF」） |
| 刪除分類 | 須確認後刪除，不影響已記錄的交易 |

#### 4.4.4 交易記錄

| 功能 | 描述 |
|------|------|
| 手動新增交易 | 選擇買/賣方向、日期、分類、股票代號、股數、價格 |
| 批次匯入 | 上傳 CSV/Excel 檔案，自動識別中英文欄位名稱 |
| 觀察清單 | 新增「WATCH」類型交易（0 股數），放入觀察名單追蹤 |
| 交易流水帳 | 完整歷史記錄，按時間倒序排列 |

**批次匯入支援欄位：**

| 中文欄位 | 英文欄位 | 資料型態 |
|---------|---------|---------|
| 股票代號 | Symbol | 文字 |
| 買賣 | Type | BUY / SELL |
| 股數 | Shares | 正整數 |
| 價格 | Price | 正數 |
| 日期 | Date | YYYY-MM-DD |

**批次匯入驗證規則：**
- `shares` 必須 > 0
- `price` 必須 ≥ 0
- `type` 必須為 BUY、SELL、WATCH 其中之一
- 不符合規則的行自動跳過，回報成功匯入筆數

#### 4.4.5 持倉展示

| 欄位 | 說明 |
|------|------|
| 股票代號 | 大寫顯示 |
| 股數 | 目前持有股數（FIFO 計算） |
| 平均成本 | 加權平均買入成本 |
| 即時報價 | 從 Yahoo Finance 取得 |
| 未實現損益 | 顏色標示（綠 = 獲利，紅 = 虧損） |

#### 使用者故事

- 作為投資者，我希望上傳 Excel 記錄後，系統自動識別欄位並匯入交易，不需要手動逐筆輸入。
- 作為投資者，我希望看到每個分類的小計與總帳戶績效，判斷哪個策略最有效。

### 4.5 自訂資料源 (Custom Feed Manager)

**目的：** 訂閱並彙整個人化財經資訊來源。

#### 功能項目

| 功能 | 描述 |
|------|------|
| 新增 RSS 源 | 輸入名稱 + RSS URL，取得最新 5 則文章 |
| 新增 HTML 頁面 | 輸入名稱 + 網頁 URL，抓取 OpenGraph 標題與縮圖 |
| 刪除資料源 | 即時移除，下次聚合不再包含 |
| 時間線展示 | 所有來源按發布時間倒序排列，顯示標題、來源名稱、時間 |
| 請求超時保護 | 每個來源 10 秒 timeout，避免慢速來源拖慢整體 |
| 防止重複訂閱 | 相同 URL 只能新增一次（資料庫唯一約束） |

#### 支援來源類型

| 類型 | 解析方式 | 取得資料 |
|------|---------|---------|
| RSS | RSS Parser 解析 | 標題、連結、發布日期 |
| HTML | Cheerio 抓取 OG Tags | 標題（og:title）、縮圖（og:image）、頁面連結 |

### 4.6 Google Sheets 自選股清單 (Watchlist via Google Sheets)

**目的：** 串接使用者現有的 Google Sheets 自選股清單，無縫整合到儀表板。

#### 功能項目

| 功能 | 描述 |
|------|------|
| 連結 Sheet | 貼入 Google Sheet ID 或完整 URL，系統自動提取 ID |
| 公開試算表 | 透過「發布至網路」CSV 匯出，無需登入 |
| 私有試算表 | 透過 Service Account 存取，需設定環境變數 |
| 自動欄位識別 | 自動找到「代號」或「Symbol」欄位作為股票代號來源 |
| 自動刷新 | 每 60 秒自動更新試算表資料 |
| 百分比彩色標示 | 自動識別百分比欄位，正數綠、負數紅 |

#### 個股功能（每列）

| 按鈕 | 功能 |
|------|------|
| 分析 | 開啟技術分析 Modal，顯示趨勢、SMA、RSI(14)、MACD、近期新聞 |
| 加入 | 快速加入 Portfolio，選擇「觀察」或「買入」模式 |

#### 批次功能

| 功能 | 描述 |
|------|------|
| 觀察清單整體健檢 | 一次分析最多 10 支股票，依技術強弱分類為三等級 |
| 每週 AI 健檢報告 | 產生完整週報，可手動觸發或設定排程，可廣播至 Discord |
| 同步至 Portfolio | 將 Sheet 中的股票批次加入 Portfolio 觀察清單 |

### 4.7 AI 投資分析功能

#### 4.7.1 AI Chat — 投資小老師

**目的：** 雙 AI 模型並行分析，提供個股快速概覽。

| 項目 | 規格 |
|------|------|
| 輸入 | 股票代號或中文名稱 |
| AI 模型 | OpenAI GPT-4o + Google Gemini 2.5 Flash（並行） |
| 語言 | 繁體中文 |
| 輸出格式 | 概念分類 / 即時數據解讀 / 近期焦點 / 分析師評語 |
| 限制 | 不提供明確買賣建議（符合法規） |

#### 4.7.2 Community Assistant — 社群小助手

**目的：** 分析社群截圖中的財經情報。

| 項目 | 規格 |
|------|------|
| 輸入 | 圖片（JPG/PNG，最大 5MB） |
| AI 模型 | OpenAI GPT-4o Vision |
| 輸出 | 提取股票代號、情報類型、客觀評語 |
| 無情報時 | 顯示「未偵測到明確財經情報」 |

#### 4.7.3 個股買入評估

**目的：** 快速給出技術面買入訊號。

| 項目 | 規格 |
|------|------|
| 觸發 | 觀察清單的「分析」按鈕 |
| 計算指標 | 5MA、20MA、60MA 排列；RSI(14)；成交量突增（> 2x 均量） |
| 輸出格式 | 🟢 買入佈局 / 🟡 繼續觀察 / 🔴 暫時迴避 + 2-3 句說明 |
| AI 模型 | GPT-4o-mini（主），Gemini 2.5 Flash（備援） |

#### 4.7.4 每週自選股 AI 健檢報告

**目的：** 自動化週報，一次掌握整個觀察清單狀況。

| 項目 | 規格 |
|------|------|
| 觸發方式 | 手動按鈕 或 Vercel Cron 定時排程 |
| 分析股數 | 最多 10 支（避免 API 超時） |
| 報告語言 | 繁體中文 |
| 分類結果 | 🟢 強勁趨勢 / 🟡 盤整觀察 / 🔴 弱勢警告 |
| 廣播 | 可選擇性設定 Discord Webhook 自動發送 |

### 4.8 Discord 機器人 — 情報員雷達

**目的：** 在投資社群 Discord 頻道中，自動過濾並分析財經情報。

| 功能 | 描述 |
|------|------|
| 文字分析 | 監聽所有非機器人訊息，判斷是否含財經情報 |
| 圖片分析 | 支援含圖片的訊息，使用 GPT-4o Vision 解讀 |
| 過濾雜訊 | 日常閒聊回傳 NULL，不回覆，避免洗版 |
| 有情報時 | 自動 Reply 提取到的股票名稱、多空看法與評語 |
| 防止熱重載多開 | 使用 global 變數保留 Discord Client 實例 |

## 5. 非功能需求 (Non-Functional Requirements)

### 5.1 效能

| 指標 | 目標 |
|------|------|
| 指數報價載入 | 4 支指數並行請求，< 3 秒完成 |
| 個股分析 API | < 5 秒（Yahoo Finance 歷史資料取得） |
| Portfolio 載入 | < 5 秒（含所有持股並行報價） |
| 自訂 Feed 聚合 | 每個來源 10 秒 timeout，整體 < 15 秒 |

### 5.2 可靠性

| 項目 | 要求 |
|------|------|
| AI 備援 | 主模型失敗時自動切換備援模型 |
| RSS 錯誤隔離 | 單一 RSS 源失敗不影響其他源 |
| 股票代號找不到 | 回傳友善錯誤訊息，不中斷整個頁面 |

### 5.3 安全性

| 項目 | 要求 |
|------|------|
| API 金鑰管理 | 所有金鑰透過環境變數管理，不寫入程式碼 |
| Input 驗證 | 所有 API 入口驗證必填參數，缺少時回傳 400 |
| 批次匯入 | 驗證每筆資料格式與數值範圍，拒絕不合法資料 |

### 5.4 資料庫

| 項目 | 要求 |
|------|------|
| ORM | Prisma |
| 資料庫 | PostgreSQL（生產環境） |
| 索引 | Trade(symbol)、Trade(date)、Trade(categoryId)、AccountTransaction(accountId) |
| 唯一約束 | DataSource.url、PortfolioCategory.name、SystemSetting.key |

## 7. 資料模型摘要 (Data Model Summary)

```
Trade           — 買賣記錄（symbol, type, shares, price, date, categoryId, accountId）
PortfolioCategory — 投資分類（name）
Account         — 模擬帳戶（balance, totalDeposit）
AccountTransaction — 帳戶流水帳（type, amount, notes）
DataSource      — 自訂資料源（name, url, type）
NetWorthSnapshot — 淨值快照（date, totalValue）
SystemSetting   — 系統設定（key, value，如 GOOGLE_SHEET_ID）
```
